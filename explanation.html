Explanation

Let the probability you pick be denoted as \(p\). For a fair coin, this probability in decimal form would be \(p = 0.5\). We can convert this decimal into binary form such that the \(i^{th}\) binary digit is given by \(b_i\). More formally,

$$ {\exists (b_1, b_2,..., b_n) \mid (\forall i \in \text{{1, n} } b_i \in \text{{0, 1}}) \land (p = 0.b_1b_2b_3...b_n)}. $$

From here, the procedure is simple:

First, designate a side you plan to stop on. For our purposes, this side is designated to be heads.

Then, continue flipping a fair coin until you land on heads.

Suppose you land on heads on the kth flip. If \(b_k = 1\), then return a successful flip. Otherwise, return that the flip was unsuccessful.



To see that this emulates any probability, note the probability of a successful flip is given by

$$ {P\text{[Successful Flip]} = P\text{[Flip 1 is first heads]}P[b_1 = 1] + ... + P\text{[Flip n is first heads]}P[b_n = 1]}. $$

Note that the probability of flip \(k\) being the first heads is \(\frac{1}{2^k}\) and the probability that \(b_k = 1\) is simply given by \(b_k\)(either 0 or 1). Therefore, the probability of a successful flip can be expressed as

$$ {P\text{[Successful Flip]} = \frac{b_1}{2} + ... + \frac{b_n}{2^n},} $$

which is the binary expansion of \(p\). Therefore, the probability of our simulation to return a successful flip is given by \(p\).


Note that this simulator uses a pseudorandom number generator, which may occasionally return skewed results.
